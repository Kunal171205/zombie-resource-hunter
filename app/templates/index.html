FinOps Knowledge Center

What is a Zombie?

A cloud resource that is "running but idle". Like a lightbulb left on in an empty room, it wastes money every second it's active.

Unattached EBS

When instances are deleted, their disks (EBS) are often left "Available". AWS still charges full price! Use "Delete on Termination" to automate.

Orphaned Snapshots

Backups of old, deleted volumes are "ghost" costs. Clean up snapshots that are older than 30 days if not needed for compliance.

Idle Elastic IPs

AWS charges a penalty for static IPs that are NOT attached to running servers. This prevents "camping" on public IP addresses.

Zombie ELBs

Load Balancers with zero healthy targets or zero traffic still cost ~$15/month. If it has no instances, it's a zombie!

FinOps Strategy

🚀 Upgrade to gp3: Save up to 20% compared to gp2 volumes with better performance.

⏰ Schedule Stops: Use "Instance Scheduler" to turn off dev boxes during weekends.

📏 Right-sizing: Move oversized instances to smaller families (e.g., from t3.large to t3.medium).
